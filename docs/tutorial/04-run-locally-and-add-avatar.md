# Chapter 4 — 봇 실행해보기 + 아바타 추가

> 10~15분 걸려요. 이번 챕터가 끝나면 **내 컴퓨터 브라우저에서 봇이 실제로 움직이는 모습**을 보게 됩니다.

---

## 왜 이걸 하는가?

지금까지는 코드만 받았어요. 이제 **실행**해 볼 차례입니다. 그리고 3D 캐릭터(아바타)를 추가해서 봇에 얼굴을 만들어줄 거예요.

이번 챕터에서 할 일:
1. 봇 실행에 필요한 부품들 다운로드 (`npm install`)
2. 봇 서버 켜기 (`npm run dev`)
3. 브라우저로 접속해서 봇 화면 보기
4. 아바타(VRM 파일) 추가해서 캐릭터 살리기

---

## Step 1. `npm install` — 부품 다운로드

CMD를 열고 봇 폴더로 이동:
```cmd
cd C:\dev\my-bot
```

이제 다음 명령어 입력:
```cmd
npm install
```

> 💡 `npm install`은 "이 프로젝트가 필요로 하는 자바스크립트 부품들(라이브러리)을 자동으로 다운받아라"는 뜻이에요. 약 300개 이상의 부품이 다운로드됩니다.

### ⚠ 여기서 학생들이 가장 많이 막힙니다

**1~3분 동안 화면이 안 움직이는 것처럼 보일 수 있어요.** 멈춘 것 같지만 정상입니다. 인터넷에서 부품들을 받고 있어요.

```
npm warn deprecated ...
npm warn deprecated ...
(한참 멈춤... 1분~3분)
```

> 🛟 **절대 Ctrl+C 누르지 마세요.** 그냥 기다리세요. 봉주도 처음에는 "멈춘 줄 알고" Ctrl+C 눌렀다가 처음부터 다시 했어요.

성공 시 마지막에 이런 출력이 나옵니다:
```
added 387 packages, and audited 388 packages in 2m

87 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

(숫자는 다를 수 있고, "vulnerabilities" 경고가 몇 개 떠도 무시해도 OK.)

이제 `dir` 해보면 `node_modules` 라는 새 폴더가 생긴 걸 볼 수 있어요. 이게 다운받은 부품들이에요.

---

## Step 2. `npm run dev` — 봇 서버 켜기

같은 CMD 창에서 이어서 입력:
```cmd
npm run dev
```

몇 초 후 이런 출력이 나옵니다:
```
> cha-bot-starter-kit@1.0.0 dev
> vite

  VITE v8.0.3  ready in 1234 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

**`http://localhost:5173/` 가 보이면 성공!** 봇이 내 컴퓨터 안에서 돌아가고 있다는 뜻이에요.

### ⚠⚠ 매우 중요 — 이 CMD 창을 **닫지 마세요**

> 봉주도 여기서 한 번 실수했습니다.

이 CMD 창은 **봇 서버가 살아있는 동안 계속 켜둬야** 합니다. 닫으면 봇도 꺼져요.

**또 다른 명령어를 쳐야 한다면 → 새 CMD 창을 따로 여세요.** (Step 4에서 그렇게 합니다.)

---

## Step 3. 브라우저에서 봇 화면 열기

브라우저(Chrome, Edge, Firefox 무엇이든) 주소창에 입력:
```
http://localhost:5173
```

이런 화면이 나오면 성공입니다:

```
┌──────────────────────────────────────┐
│                                      │
│         👋 아바타가 비어있어요         │
│                                      │
│    public/avatar.vrm 파일을 추가하면   │
│      여기에 캐릭터가 나타납니다.        │
│                                      │
└──────────────────────────────────────┘
```

> 💡 이 친절한 안내문은 "아직 아바타 파일이 없다"는 뜻이에요. 다음 단계에서 추가하면 사라지고 진짜 캐릭터가 등장합니다.

### 화면이 안 나올 때

- **빈 화면**: 페이지 새로고침 (F5) 한 번 해보세요.
- **"이 사이트에 연결할 수 없음"**: CMD 창에서 `npm run dev` 출력이 정상인지 다시 확인. 에러 메시지가 있다면 그걸 캡처해서 박교수님이나 봉주에게 보여주세요.
- **포트 충돌 에러** (`Port 5173 is already in use`): 다른 프로그램이 같은 포트를 쓰고 있어요. CMD에서 Ctrl+C 로 끄고 다시 `npm run dev` 하면 보통 다른 포트(5174 등)로 자동 변경됩니다.

---

## Step 4. 아바타(VRM 파일) 추가하기

> **이 챕터에서 가장 헷갈리는 부분입니다.** 봉주도 처음에 잘못된 폴더에 넣어서 헤맸어요. 그래서 **CMD 명령어로 넣는 방법**을 추천합니다 — 절대 실패하지 않는 방법이에요.

### 4-1. 아바타 파일(`.vrm`) 구하기

먼저 VRM 파일이 있어야 해요. 두 가지 방법:

**방법 A. VRoid Hub에서 다운로드 (가장 쉬움) ⭐**
1. [https://hub.vroid.com](https://hub.vroid.com) 접속
2. 마음에 드는 캐릭터 클릭
3. **다운로드** 가능한 캐릭터인지 확인 (모든 캐릭터가 다운로드 가능한 건 아님)
4. 다운로드 → `.vrm` 파일이 다운로드 폴더에 저장됨

**방법 B. VRoid Studio로 직접 만들기**
1. [https://vroid.com/studio](https://vroid.com/studio) 에서 VRoid Studio 다운로드 (무료)
2. 설치 후 실행 → 캐릭터 만들기
3. 처음이라면 **AvatarSample_A** 템플릿으로 시작하면 빠릅니다
4. **Export → VRM** 으로 내보내기

> ⚠ **License 경고 — VRoid Studio Export 시 반드시 확인하세요:**
> 기본값은 제약이 많아요. 자신이 만든 캐릭터라도 다음과 같이 설정하시면 자유롭게 쓸 수 있어요:
> - `Avatar Permission`: **everyone**
> - `Commercial Use`: **allow**
> - `Redistribution`: **allow**
> - `Modification`: **allow**
>
> 이 부분은 Chapter 5(또는 별도 챕터)에서 더 자세히 다룰 예정입니다.

받은 `.vrm` 파일은 일단 **바탕화면**에 두세요. (이름이 예를 들어 `AvatarSample_A.vrm` 이라고 가정)

### 4-2. 아바타 파일을 `public/` 폴더에 넣기 — CMD 방법 (추천)

> ⭐ 이게 가장 안전하고 빠른 방법입니다.

**새 CMD 창**을 엽니다 (기존 `npm run dev` 창은 그대로 두고).

다음 두 줄을 입력하세요:

```cmd
cd C:\dev\my-bot\public
copy "%USERPROFILE%\Desktop\AvatarSample_A.vrm" avatar.vrm
```

> 💡 `%USERPROFILE%\Desktop` 은 "내 바탕화면"의 자동 경로예요. `AvatarSample_A.vrm` 부분은 본인이 받은 파일 이름에 맞게 바꾸세요.

성공 시 출력:
```
        1개 파일이 복사되었습니다.
```

이게 핵심입니다: **파일 이름은 반드시 `avatar.vrm`** (소문자, 정확히 이 이름) — 봇이 이 이름을 찾도록 코딩되어 있어요.

### 4-3. (백업 방법) 파일 탐색기 드래그앤드롭

만약 위 CMD 방법이 안 되면 이걸 시도:

1. 파일 탐색기 열기 (윈도우 키 + E)
2. 주소창에 입력: `C:\dev\my-bot\public` (엔터)
3. 바탕화면에 있는 `.vrm` 파일을 이 폴더로 드래그
4. 복사된 파일을 우클릭 → 이름 바꾸기 → `avatar.vrm` (확장자 포함, 정확히)

> ⚠ **여기서 봉주도 헷갈렸습니다**: 파일 탐색기로 옮기다 보면 `public` 옆에 있는 다른 폴더 (`src` 등)에 실수로 떨어뜨리기 쉬워요. **CMD 방법이 훨씬 안전**합니다.

### 4-4. 브라우저 새로고침

이제 브라우저로 돌아가서 **강력 새로고침**:
- Windows: **Ctrl + Shift + R**
- Mac: **Cmd + Shift + R**

> 💡 일반 새로고침(F5)으로는 캐시 때문에 이전 화면이 그대로 보일 수 있어요. **Ctrl + Shift + R** 로 강제 새로고침해야 새 아바타 파일을 읽습니다.

**🎉 캐릭터가 등장하면 성공!**
- 자연스럽게 눈을 깜빡이고
- 가슴이 호흡하듯 위아래로 움직이고
- 가만히 서있는 idle 애니메이션을 합니다

## 확인하기

다음 두 가지가 모두 OK면 성공:

1. ✅ 첫 번째 CMD 창: `npm run dev` 가 계속 실행 중이고 에러 없음
2. ✅ 브라우저: `http://localhost:5173` 에서 캐릭터가 움직이고 있음 (또는 빈 아바타 안내문이 사라짐)

---

## 일을 마치고 끄는 법

오늘 작업을 끝내고 컴퓨터를 끄거나, 잠깐 쉬려면:

1. `npm run dev` 가 돌아가는 CMD 창에서 **Ctrl + C** 누르기
2. "일괄 작업을 끝내시겠습니까 (Y/N)?" 물어보면 **Y** 입력 → 엔터
3. CMD 창 닫기

다음에 다시 시작할 때는:
```cmd
cd C:\dev\my-bot
npm run dev
```
만 다시 치면 됩니다. (`npm install` 은 한 번만 하면 돼요.)

## 🛟 자주 묻는 질문

**Q. `npm install`이 한참 멈춰있어요. 정말 정상인가요?**
> 네, 매우 흔한 일입니다. 인터넷 속도와 컴퓨터 성능에 따라 1~5분 걸려요. **Ctrl+C 누르지 말고 그냥 기다리세요.** 5분 넘어가도 응답이 없으면 그때 Ctrl+C 후 다시 `npm install`.

**Q. `npm run dev` 출력에 빨간 글씨로 경고가 떠요. 망한 건가요?**
> "Warning"은 무시해도 됩니다. "Error"라고 적힌 빨간 글씨가 아니라면 정상이에요. `Local: http://localhost:5173/` 줄만 나오면 성공입니다.

**Q. 아바타를 넣었는데 안 보여요. 빈 화면이거나 안내문이 그대로예요.**
> 체크리스트:
> 1. 파일 이름이 정확히 `avatar.vrm` 인가요? (대소문자 구분, `Avatar.vrm` 도 안 됨)
> 2. 위치가 정확히 `C:\dev\my-bot\public\avatar.vrm` 인가요? (`src/` 폴더에 넣으면 안 됩니다)
> 3. 브라우저에서 **Ctrl+Shift+R** 강제 새로고침 했나요?
>
> CMD에서 확인:
> ```cmd
> dir C:\dev\my-bot\public\avatar.vrm
> ```
> 파일이 보이면 위치는 OK.

**Q. localhost:5173 이 아닌 다른 포트로 나와요 (예: 5174).**
> 정상입니다. 5173이 이미 다른 프로그램에 의해 점유되어 있으면 자동으로 다음 번호로 바뀌어요. CMD에 출력된 그 주소를 그대로 브라우저에 입력하세요.

**Q. 캐릭터가 너무 크거나 작아요. 위치도 이상해요.**
> 아바타마다 기본 크기가 달라서 그럴 수 있어요. 다음 챕터들에서 카메라 위치/줌 조정 방법을 다룰 예정입니다. 일단은 "보이기만 하면" 성공입니다.

**Q. VRM 파일 어디서 받으면 좋을지 더 추천해주세요.**
> - [VRoid Hub](https://hub.vroid.com) — 가장 큰 라이브러리
> - [Booth](https://booth.pm) — 유료/무료 캐릭터 마켓 (일본 사이트)
> - VRoid Studio로 직접 만들기 — 완전한 본인 캐릭터를 원한다면 ⭐ 추천

**Q. 다른 사람이 만든 VRM을 봇에 써도 되나요?**
> **반드시 라이선스 확인이 필요합니다.** 캐릭터 만든 사람이 "타인의 사용 허용", "상업적 이용 허용" 등을 명시했는지 봐야 해요. VRoid Hub에서는 각 캐릭터 페이지 하단에 라이선스 표시가 있어요. 다음 챕터에서 더 자세히 다룹니다.

**Q. Mac/Linux 사용자입니다.**
> 명령어 거의 동일합니다. 파일 복사 부분만 다음과 같이 바꾸세요:
> ```bash
> cd ~/dev/my-bot/public
> cp ~/Desktop/AvatarSample_A.vrm avatar.vrm
> ```

**Q. 봉주처럼 잘못된 폴더에 넣어서 헷갈렸을 때 어떻게 정리하나요?**
> 잘못 넣은 파일을 그 폴더에서 삭제하고, 위의 CMD 방법으로 다시 시도하세요. 잘못된 파일이 남아있어도 봇 동작에 영향은 없지만, 폴더가 어지러워지면 나중에 헷갈리니까 정리하는 게 좋아요.
